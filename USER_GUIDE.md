## 应用介绍

LogMonitor 是一款跨平台日志监控桌面应用，基于 Electron + React 构建。它可以帮助您实时监控多个日志文件，自动解析常见日志格式，并提供强大的搜索过滤和告警功能。

### 适用场景

- 🖥️ 开发环境：监控应用日志，快速定位问题
- 🚀 生产环境：实时监控系统日志，及时发现异常
- 📊 运维工作：集中管理多台服务器日志
- 🔍 调试分析：搜索和过滤特定日志内容

### 系统要求

| 系统 | 最低要求 | 推荐配置 |
|------|----------|----------|
| Windows | Windows 10 x64 | Windows 11 x64 |
| Linux | Ubuntu 18.04+ | Ubuntu 22.04+ |
| 内存 | 4 GB | 8 GB |
| 磁盘 | 500 MB | 1 GB |
| Node.js | 18.x | 20.x |

## 功能说明

### 核心功能

| 功能 | 说明 |
|------|------|
| 📊 **实时监控** | 同时监控多个日志文件，实时捕获新增日志 |
| 🔍 **智能解析** | 自动识别 7+ 种常见日志格式，无需手动配置 |
| 🎯 **过滤搜索** | 按级别、关键字、时间范围多维度筛选 |
| ⚠️ **告警通知** | ERROR/FATAL 级别或自定义关键字触发告警 |
| 📈 **统计分析** | ECharts 可视化图表，直观展示日志趋势 |
| ⚙️ **配置管理** | 可视化配置界面，支持导入导出 |

### 支持的日志格式

LogMonitor 自动识别以下日志格式：

1. **括号格式**
   ```
   [2024-01-01 12:00:00] [ERROR] 数据库连接失败
   ```

2. **ISO 格式**
   ```
   2024-01-01T12:00:00.000Z ERROR 数据库连接失败
   ```

3. **简单格式**
   ```
   2024-01-01 12:00:00 ERROR 数据库连接失败
   ```

4. **JSON 格式**
   ```json
   {"timestamp": "2024-01-01T12:00:00.000Z", "level": "ERROR", "message": "数据库连接失败"}
   ```

5. **Apache 访问日志**
   ```
   127.0.0.1 - - [01/Jan/2024:12:00:00 +0800] "GET /api/users HTTP/1.1" 200 1234
   ```

6. **Syslog 格式**
   ```
   Jan  1 12:00:00 server01 nginx[1234]: Connection timeout
   ```

### 日志级别支持

| 级别 | 说明 | 颜色 |
|------|------|------|
| DEBUG | 调试信息 | 灰色 |
| INFO | 普通信息 | 绿色 |
| WARN | 警告信息 | 黄色 |
| ERROR | 错误信息 | 红色 |
| FATAL | 致命错误 | 深红色 |

## 安装步骤

### Windows 安装

#### 方法一：安装包安装（推荐）

1. 下载安装包 `LogMonitor Setup x.x.x.exe`
2. 双击运行安装程序
3. 选择安装路径（默认 `E:\LogMonitor\`）
4. 点击"安装"完成安装
5. 在开始菜单或桌面快捷方式启动应用

#### 方法二：源码安装

1. 安装 Node.js（https://nodejs.org/）
2. 验证安装：
   ```bash
   node --version
   npm --version
   ```
3. 下载项目源码
4. 打开命令提示符，进入项目目录：
   ```bash
   cd E:\openclaw-projects\log-monitor-app
   ```
5. 安装依赖：
   ```bash
   npm install
   ```
6. 初始化数据目录：
   ```bash
   node scripts/init-data-dir.js
   ```
7. 启动应用：
   ```bash
   npm start
   ```

### Linux 安装

#### 方法一：AppImage（推荐）

1. 下载 `LogMonitor-x.x.x.AppImage`
2. 赋予执行权限：
   ```bash
   chmod +x LogMonitor-x.x.x.AppImage
   ```
3. 运行应用：
   ```bash
   ./LogMonitor-x.x.x.AppImage
   ```

#### 方法二：DEB 包（Debian/Ubuntu）

1. 下载 `log-monitor-app_x.x.x_amd64.deb`
2. 安装：
   ```bash
   sudo dpkg -i log-monitor-app_x.x.x_amd64.deb
   ```
3. 启动应用：
   ```bash
   logmonitor
   ```

#### 方法三：源码安装

1. 安装 Node.js：
   ```bash
   curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
   sudo apt-get install -y nodejs
   ```
2. 下载项目源码
3. 安装依赖：
   ```bash
   cd log-monitor-app
   npm install
   ```
4. 初始化数据目录：
   ```bash
   node scripts/init-data-dir.js
   ```
5. 启动应用：
   ```bash
   npm start
   ```

## 快速开始

### 首次使用

1. **启动应用**
   - Windows: 双击桌面快捷方式或从开始菜单启动
   - Linux: 运行 `./LogMonitor.AppImage` 或 `logmonitor`

2. **进入监控配置页面**
   - 点击顶部导航栏的"监控配置"

3. **创建第一个监控任务**
   - 点击"新建监控"按钮
   - 填写监控名称（如"应用日志"）
   - 添加日志文件路径
   - 设置告警级别
   - 点击"创建监控"

4. **启动监控**
   - 在监控列表中找到刚创建的监控
   - 点击"启动"按钮
   - 状态变为"运行中"

5. **查看日志**
   - 点击"日志查看"页面
   - 实时日志开始显示

## 配置监控任务

### 创建监控任务

1. 点击顶部导航的 **"监控配置"**

2. 点击 **"新建监控"** 按钮

3. 填写监控信息：

   | 字段 | 说明 | 必填 |
   |------|------|------|
   | 监控名称 | 自定义监控任务名称 | 是 |
   | 监控路径 | 要监控的日志文件路径 | 是 |
   | 告警级别 | 触发告警的日志级别 | 是 |
   | 自定义关键字 | 额外触发告警的关键字 | 否 |
   | 描述 | 监控任务描述 | 否 |

4. 点击 **"创建监控"** 保存

### 添加监控路径

支持两种方式添加日志文件：

#### 选择文件
- 点击"选择文件"按钮
- 在文件选择器中选择一个或多个日志文件
- 支持多选（Ctrl/Cmd + 点击）

#### 选择文件夹
- 点击"选择文件夹"按钮
- 选择一个目录
- 自动监控该目录下所有 `.log` 文件

### 配置告警规则

#### 日志级别告警

默认告警级别：
- ✅ ERROR - 错误
- ✅ FATAL - 致命错误
- ✅ CRITICAL - 严重错误

可选级别：
- ⬜ WARN - 警告
- ⬜ INFO - 信息
- ⬜ DEBUG - 调试

#### 关键字告警

添加需要特别关注的关键字：
```
exception
failed
timeout
out of memory
connection refused
```

### 编辑监控任务

1. 在监控列表中找到目标监控
2. 点击"编辑"按钮
3. 修改配置信息
4. 点击"保存"

### 删除监控任务

1. 在监控列表中找到目标监控
2. 点击"删除"按钮
3. 确认删除

> ⚠️ **注意**: 删除监控任务不会删除已收集的日志数据

### 启动/停止监控

| 操作 | 方法 |
|------|------|
| 启动监控 | 点击监控列表中的"启动"按钮 |
| 停止监控 | 点击监控列表中的"停止"按钮 |
| 重启监控 | 先停止，再启动 |

## 查看日志和搜索

### 日志查看页面

进入 **"日志查看"** 页面，可以看到：

- 📋 日志列表：按时间倒序显示所有日志
- 🔍 搜索框：快速搜索日志内容
- 🎚️ 过滤器：按条件筛选日志
- 📄 分页器：浏览更多日志

### 搜索功能

#### 关键字搜索

在搜索框中输入关键字：
```
数据库连接
```

支持多关键字（空格分隔）：
```
数据库 连接失败
```

#### 精确搜索

使用引号进行精确匹配：
```
"Connection timeout"
```

### 过滤功能

#### 按级别过滤

点击级别按钮筛选：
- 🔴 ERROR
- 🟡 WARN
- 🟢 INFO
- ⚪ DEBUG
- ⚫ FATAL

支持多选，点击多个级别同时筛选。

#### 按时间范围过滤

1. 点击"时间范围"按钮
2. 选择开始时间和结束时间
3. 点击"应用"

预设时间范围：
- 最近 1 小时
- 最近 6 小时
- 最近 24 小时
- 最近 7 天
- 自定义范围

#### 按来源文件过滤

1. 点击"来源"下拉框
2. 选择日志文件
3. 只显示该文件的日志

### 日志详情

点击任意日志行，查看详细信息：

```
┌─────────────────────────────────────┐
│ 日志详情                             │
├─────────────────────────────────────┤
│ 时间：2024-01-01 12:00:00           │
│ 级别：ERROR                         │
│ 来源：application.log               │
│ 内容：数据库连接失败                │
│ 原始：[2024-01-01 12:00:00]...      │
└─────────────────────────────────────┘
```

### 导出日志

1. 筛选出需要的日志
2. 点击"导出"按钮
3. 选择格式：
   - CSV
   - JSON
   - TXT
4. 选择保存路径
5. 点击"导出"

## 告警配置

### 告警设置

进入 **"设置"** 页面，找到 **"告警配置"** 部分。

### 通知方式

#### 1. 桌面通知（默认开启）

当触发告警时，系统会弹出桌面通知。

配置选项：
- ✅ 启用通知
- 🔊 通知声音
- ⏱️ 通知持续时间

#### 2. 声音告警

配置选项：
- ✅ 启用声音
- 🔈 选择声音文件
- 🔊 音量大小

#### 3. 邮件告警（待实现）

配置选项：
- ✅ 启用邮件
- 📧 SMTP 服务器
- 📧 SMTP 端口
- 👤 用户名
- 🔑 密码
- 📨 收件人列表

#### 4. Webhook 告警（待实现）

配置选项：
- ✅ 启用 Webhook
- 🔗 Webhook URL
- 📝 请求方法（POST/GET）
- 📋 自定义请求头

### 告警规则

#### 级别告警

默认触发告警的级别：
- ERROR
- FATAL
- CRITICAL

可以自定义添加或移除级别。

#### 关键字告警

添加需要触发告警的关键字：

```
exception
failed
timeout
out of memory
connection refused
null pointer
stack overflow
```

#### 频率限制

避免告警风暴：
- 同一监控任务：最多 10 条/分钟
- 同一关键字：最多 5 条/分钟

### 告警历史

查看历史告警记录：
1. 进入"仪表盘"页面
2. 找到"告警信息"卡片
3. 点击"查看详情"

## 常见问题 FAQ

### 安装问题

#### Q1: Windows 安装时提示"未知发布者"
**A**: 这是正常的安全提示。点击"更多信息"，然后选择"仍要运行"。

#### Q2: Linux AppImage 无法运行
**A**: 需要赋予执行权限：
```bash
chmod +x LogMonitor-x.x.x.AppImage
```

#### Q3: 提示"找不到模块"
**A**: 运行以下命令重新安装依赖：
```bash
npm install
```

#### Q4: 端口 3000 被占用
**A**: 
- 方法 1: 关闭占用端口的程序
- 方法 2: 修改 `package.json` 中的端口配置

### 使用问题

#### Q5: 日志没有被监控到
**A**: 检查以下几点：
1. 监控状态是否为"运行中"
2. 日志文件路径是否正确
3. 日志文件是否有读取权限
4. 日志格式是否被支持

#### Q6: 如何监控多个日志文件
**A**: 
1. 创建监控任务时，选择多个文件
2. 或创建多个监控任务

#### Q7: 日志解析不正确
**A**: 
1. 检查日志格式是否在支持范围内
2. 查看日志原始内容是否规范
3. 如不支持，可提交格式样本到 issue

#### Q8: 如何查看支持的日志格式
**A**: 支持以下格式：
- `[2024-01-01 12:00:00] [ERROR] 消息`
- `2024-01-01T12:00:00.000Z ERROR 消息`
- `2024-01-01 12:00:00 ERROR 消息`
- JSON 格式
- Apache 访问日志
- Syslog 格式

#### Q9: 告警没有触发
**A**: 检查：
1. 告警功能是否启用
2. 告警级别配置是否正确
3. 关键字是否匹配
4. 是否触发频率限制

### 性能问题

#### Q10: 应用运行卡顿
**A**: 
1. 减少同时监控的文件数量
2. 缩短日志保留天数
3. 增加刷新间隔
4. 清理缓存数据

#### Q11: 日志显示延迟
**A**: 
1. 检查日志文件大小（大文件解析较慢）
2. 调整刷新间隔（设置页面）
3. 减少同时显示的日志数量

### 数据问题

#### Q12: 配置文件在哪里
**A**: 
- Windows: `E:\LogMonitor\data\config.json`
- Linux: `~/.logmonitor/data/config.json`

#### Q13: 如何备份配置
**A**: 
1. 进入"设置"页面
2. 点击"导出配置"
3. 保存配置文件

#### Q14: 如何恢复配置
**A**: 
1. 进入"设置"页面
2. 点击"导入配置"
3. 选择备份的配置文件

#### Q15: 日志数据占用太多空间
**A**: 
1. 进入"设置"页面
2. 调整"日志保留天数"
3. 点击"清理旧日志"

### 其他问题

#### Q16: 支持监控远程日志吗
**A**: 当前版本仅支持本地文件。远程日志监控功能计划中。

#### Q17: 可以监控旋转日志吗（如 app.log.1, app.log.2）
**A**: 支持。添加监控路径时选择文件夹，会自动监控所有匹配的日志文件。

#### Q18: 如何反馈问题或建议
**A**: 
- GitHub Issues: 提交问题报告
- 邮件：support@openclaw.com
- 社区论坛：forum.openclaw.com
